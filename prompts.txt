Descripción de la UI (Párrafo Único – Adaptado a Messer)

La pantalla de autenticación será un prototipo de alta fidelidad adherido estrictamente al Tema Oscuro del sistema de diseño de Messer, creado 
bajo principios de minimalismo digital para garantizar seguridad, claridad y una interacción fluida incluso en ambientes de 
restaurante. El diseño seguirá el enfoque mobile-first, comenzando con la visualización del logotipo de Messer en el encabezado 
para reforzar la identidad de marca y generar confianza. El contenido central estará orientado a la autenticación, presentando un 
título de bienvenida y dos campos de entrada delineados (textField.outlined) para el Usuario/Correo o Número de Mesa y la Contraseña. 
Justo debajo del campo de contraseña se integrará un Toggle Switch para la función “Recordarme”, alineado con los controles del 
formulario y cumpliendo con el tamaño de objetivo táctil mínimo de 48x48 dp para mayor accesibilidad. La acción de acceso se completará
 con un botón principal de relleno (button.filled) con forma de píldora (shape.corner.full) ubicado cerca del borde inferior para 
 facilitar el alcance con una sola mano, acompañado de un enlace de soporte para recuperación de cuenta o asistencia rápida.





 Indicación Detallada de la Interfaz de Usuario (Prompt Estructurado para IA – Adaptado a Messer)

Utilizaremos la estructura de prompt recomendada (similar a ASPECCT o CTF) para garantizar que la herramienta de IA (Stitch, Figma Make, UX Pilot) genere un resultado de alta fidelidad:

Rol:
Actúa como un diseñador UX/UI Senior, especializado en aplicaciones mobile-first con el Messer Design System (MDS).

Tarea:
Generar el prototipo de alta fidelidad para la pantalla de Inicio de Sesión y Autenticación de la aplicación móvil Messer.

Contexto y Restricciones:

Tema: Usar el Tema Oscuro (dark) del MDS, optimizado para ambientes de restaurante con iluminación tenue.

Tokens: Referenciar todos los elementos con los tokens de MDS (por ejemplo: shape.corner.full, color.dark.primaryContainer, text.title.large).

Accesibilidad: Asegurar que todos los elementos táctiles, incluido el Toggle Switch, cumplan con el tamaño mínimo de 48x48dp para interacción segura y cómoda.

Espaciado: Utilizar el token de espaciado spacing.m (16dp) para la separación vertical de los bloques principales.

Identidad de marca: El logo de Messer debe aparecer destacado en el encabezado para reforzar el reconocimiento de marca.

Estructura de Componentes de UI (de Arriba a Abajo)

Encabezado

image.logo.primary (Logo de Messer centrado).

text.title.large (Mensaje de bienvenida o nombre de la app).

Formulario de Autenticación

textField.outlined – Usuario/Correo o Número de Mesa.

textField.outlined – Contraseña (con opción de mostrar/ocultar).

toggle.switch – Recordarme (alineado a la derecha del bloque de contraseña).

Acciones Primarias

button.filled – Iniciar Sesión (forma de píldora shape.corner.full).

text.link – ¿Olvidaste tu contraseña? (enlace de soporte).

button.outlined (opcional) – Ingresar como invitado si el restaurante permite pedidos sin registro.

Mensajes de Estado

text.helper para mostrar errores de validación (credenciales incorrectas, conexión fallida).

Footer (Opcional)

text.caption – Mensaje legal o de privacidad, usando spacing.m para mantener la separación con el botón principal.
